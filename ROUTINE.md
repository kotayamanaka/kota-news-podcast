# 朝の深掘りニュース — 毎朝の生成ルーチン

> このファイルは毎朝6時の scheduled-task が読んで **そのまま実行する** 手順書。
> 目的：いま話題のニュース5本を選び、5W1Hで深掘りし、約15分の音声（mp3）にして Podcast 配信する。
> 作業ディレクトリ：`C:\Users\kota\Documents\Claude\kota-news-podcast`

## 全体フロー

```
候補収集（横断） → 選定（話題性ファースト・既出除外）→ 5本を深掘りリサーチ
 → 台本（5W1H・約1000字×5）→ 結合＆読み正規化 → edge-tts → mp3
 → publish.py → git push → history.json 追記
```

---

## STEP 1. 日付の確定

- 今日の日付を `YYYY-MM-DD`（JST）で確定。以降 `<DATE>` と書く。

## STEP 2. 候補ニュースの横断収集

WebSearch を **ジャンル横断**で複数本投げ、直近24〜48時間の話題を15〜20本リストアップする。最低限カバーするジャンル：

- 日本のトレンド（Yahoo!ニュース / X トレンド / はてなブックマーク / 主要紙）
- AI・テック（Hacker News / TechCrunch / buildfastwithai 等の日次まとめ）
- 経済・金融（Reuters / CNBC / Bloomberg / 日経）
- 国際・政治（Reuters / AP / Al Jazeera）
- サイエンス・医療
- **カルチャー（映画・音楽・アート）/ 日本のエンタメ / スポーツ**
- 社会現象・バズ

> 大原則：**話題性ファースト。ジャンルは絞らない。** その日いちばん話題なものを素直に拾う。日本語圏と英語圏の両方を見る。

## STEP 3. 既出の除外

- `data/history.json` を読み、過去に取り上げたトピックと**被る候補を除外**する。
- 単なる続報も除外。ただし**重大な新展開**があった場合だけ、改めて拾ってよい（その旨を台本で触れる）。

## STEP 4. 5本に絞る（スコアリング）

各候補を採点し、上位5本を選ぶ：

| 観点 | 中身 |
|---|---|
| 話題性（最重要・必須） | いま世間で本当に盛り上がっているか。ゼロなら弾く |
| 深掘り余地 | 背景・経緯・"なぜ"が豊かで、3分語れるか |
| 「実は」度 | 名前は聞くが中身を語れる人が少ない＝解説の価値 |
| 鮮度 | 直近24〜48時間 |
| 未出 | history.json と被らない |

- **多様性は弱い制約**：同一ジャンルは原則2本まで。話題が一極集中した日は超えてOK。
- `kota適合`のような恣意的な重み付けはしない（kota 指示 2026-06-14）。

選んだ5本と「なぜ選んだか」を1行ずつ、後述の commit メッセージ or ログに残す。

## STEP 5. 5本を深掘りリサーチ

各ニュースについて、観点を分けて WebSearch / WebFetch で深掘りする（「語れるようになりたい」§3 リサーチプロセス準拠）：

1. 何が起きたか（What・サマリー）
2. いつ・どこで（When / Where）
3. だれが（Who・関係者情報。立場・背景・関係を厚めに）
4. どうやって（How・仕組み）
5. なぜ（Why・**最重要。なぜを連鎖で深掘り** 2〜3問）
6. 意外と知らないポイント

リサーチ規律（厳守）：
- **仮説・推測は書かない**。「〜だろう」「〜と見られる」は禁止。事実・固有名詞・数字で。
- 出典は一次ソース・公式発表・大手メディアを優先。確認できない数値は使わない。
- 日本語・英語の両方で検索する。
- 「普通の人がこの話題で浮かべる素朴な疑問」を各本3〜5個立て、台本で答える。

## STEP 5.5. 深掘り観点のセルフチェック（当たり前回避・必須）

深掘りの「観点」がズレると価値が出ない。台本を書く前に、各ニュースで選んだ深掘りポイントを自分で点検する：

- **「そりゃそうだろ」になっていないか？** 当たり前の問い（例：「O157って危険なの？」「戦争は良くないよね」）は深掘りではない。読者はもう答えを知っている。
- **普通の賢い人が本当に引っかかる『なぜ』はどこか？** より本質的・非自明な問いに切り込む。
  - 例：コストコ総菜のO157食中毒 → ×「O157は危険か」 → ○「**なぜ検出されたのか／どこから混入したのか／なぜその総菜で起きたのか／自主回収や処分で再発は防げるのか**」（原因・経路・構造）。
- 手順：各ニュースで「当たり前の問い」と「本当に知りたい非自明な問い」を1つずつ書き出し、**後者を深掘りの軸に据える**。
- 寄せる先：原因・経路・背景・なぜ今・なぜここ・誰が得して誰が損するか・次にどうなるか。

このセルフチェックを通してから STEP 6 の台本化に進む。観点が弱いまま書き始めない。

## STEP 6. 台本化（5W1H構成・1本約1000字）

各ニュースを次の流れで、話して伝わる文章にする（proto 基準：1本=約1000字=約3分）：

```
何が起きたか（What） → いつ・どこで（When/Where）→ だれが（Who・関係者）
 → どうやって（How）→ なぜ（Why・なぜの連鎖）→ おもしろポイント再整理 → 一言で語るなら
```

文体：
- ですます調。会話寄り（「〜なんですよね」「〜という話でした」）。
- 凝った語りフックは不要（kota 指示：語りの面白さは気にしない）。素直に分かりやすく。
- 仮説・補助線・系譜論・断定キャッチコピーは書かない。

## STEP 7. 1本に結合（オープニング＋5本＋クロージング）

全体を1つのテキストにまとめる。目安：**合計約5000字（約15分）**。

- オープニング：「おはようございます。<DATE>、朝の深掘りニュースです。今日のトピックは5つ。」＋見出し5本を一言ずつ予告
- 各セグメントの頭に軽い区切り（「1本目」「2本目」…）
- クロージング：「以上、今日の5本でした。良い一日を。」

### 読み正規化（記号・英略語）

結合テキストを書く時点で、機械的に処理しておく：

- 英略語はカタカナ読みに（例：`AI`→エーアイ、`CEO`→シー・イー・オー、`API`→エー・ピー・アイ）。固有名詞内のアルファベットも読み下す。
- 記号を言葉に（`$120`→120ドル、`80%`→80パーセント、`A/B`→エー・ビー、`〜`→から）。
- 数字は基本そのままで良い（日付・金額・件数は edge-tts が概ね正しく読む）。

台本テキストは `scripts/_today.txt`（作業用・コミットしない）に保存する。

## STEP 7.5. 読み上げ用カナ台本を作る（Claude本人がふりがなを振る・必須）

**ふりがなの主役は Claude（あなた）。** 形態素解析器は意味を理解しないので読みを外す（何→ナン、火曜日→カヨウヒ）。あなたは文脈を理解して正しく読めるので、ここはあなたが責任を持つ。合成前に、結合台本(_today.txt)を**読み上げ用にカナ化**して上書き保存する。

方針：
- 漢字語は、**文脈に合う正しい読みでカタカナ/ひらがなに変換**する（例：火曜日→カヨウビ、何が→なにが、深掘り→ふかぼり、行方→ゆくえ、O157→オーいちごーなな、重なって→かさなって）。
- **固有名詞・人名・地名・作品名・企業名**は読みを確認して正しいカナに（不確かなら WebSearch で確認）。
- **数字・日付・金額・件数は数字のまま残す**（15日・2026年・220本・5億円。edge-tts が正しく読む。むしろカナ化すると壊れる）。
- 記号は言葉に（80%→80パーセント、$→ドル、〜→から）。
- こうして「漢字をほぼ含まない読み上げ用テキスト」にする。**edge-tts はカナを誤読しないので、これでエンジン誤読（深掘り→ふかくつり型）も同時に消える。**

仕上げの保険（任意）：
- `python scripts/yomi_check.py scripts/_today.txt` で漢字の残りを確認できる（数字直後の助数詞=15日 等は残してよい。synth が保護する）。
- 毎回出る固有名詞・定番語は `data/yomi_dict.json` に追記すると次回から自動置換（kota に聞かず追記してよい）。
- synth.py も「dict適用＋残った漢字の自動カナ化（数字直後の助数詞は保護）」を行うが、これは取りこぼし用の保険。**主役はあなたのカナ化。機械任せにしない。**

> 注意：edge-tts の読みは100%は予測できない。だから「不安なものはカタカナに倒す」を徹底し、辞書を育てることで、回を重ねるほど誤読を減らす。
>
> **重要な穴**：解析器(yomi_check)が正しく読む語でも、edge-tts だけが誤読することがある（例：「深掘り」→解析器は「ふかぼり」だが edge-tts は「ふかくつり」）。この型はチェックでは捕まえられず、`data/yomi_dict.json` への強制ルビ登録でしか直せない。冒頭・結びの定型句（「朝の深掘りニュース」等）の語はとくに毎回出るので、気づいたものは即辞書へ。

## STEP 8. 音声合成

```
python scripts/synth.py scripts/_today.txt episodes/<DATE>.mp3
```

（声 = Nanami 固定。rate 既定。synth.py が data/yomi_dict.json を自動適用する。）

## STEP 9. 配信（feed 更新）

**日本語をコマンドライン引数で渡さない**（Windows で文字化けするため）。タイトル・概要は必ず**ファイル経由**で渡す：

1. `scripts/_meta.json` を UTF-8 で書く：
   ```json
   { "title": "<DATE> 朝の深掘りニュース", "desc": "<5本の見出しを ／ で連結>" }
   ```
2. 引数なしで publish を実行（title/desc は _meta.json から読まれる）：
   ```
   python scripts/publish.py episodes/<DATE>.mp3 <DATE>
   ```

- これで `episodes/<DATE>.mp3` 配置、`feed.xml` 再生成、直近30本に剪定、`data/episodes.json` 更新まで完了。
- ⚠️ `publish.py ... "日本語タイトル"` のように**引数で日本語を渡すと化ける**。必ず _meta.json 経由にする。

## STEP 10. git push

```
git add -A
git commit -m "episode <DATE>: <5本の見出し要約>"
git push
```

- push 後、数分で `https://kotayamanaka.github.io/kota-news-podcast/feed.xml` に反映され、Pod アプリが自動取得する。

## STEP 11. history.json 追記

- 今日選んだ5本を `data/history.json` に追記（date / title / keywords）。重複回避のため必須。
- これも上の commit に含めてよい（STEP 10 の前に追記しておく）。
