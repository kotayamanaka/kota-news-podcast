# 外食ラジオ — 番組差分ルーチン（SHOW=gaishoku）

> 制作の本体手順は **RADIO_BASE.md** に従う。このファイルは**テーマ選定と番組の狙い**だけを定義する。
> scheduled-task：`gaishoku-radio`（毎朝2:20頃。全番組を6時までに仕上げる方針）。配信フィード：`gaishoku.xml`。
> 2026-06-28 新設（kota 指示「飲食チェーン解説ラジオ」）。番組名は推奨で「外食ラジオ」に確定。

## この番組の狙い

kota が、よく行く・気になる **外食／飲食チェーン店** を「ただの常連」から「**仕組みまで語れる**」に変えるための番組。1日1チェーンを深掘りする。

- 対象は**飲食チェーン**＝複数店舗を展開する外食ブランド。ファストフード（マクドナルド・モスバーガー・ケンタッキー）、牛丼（吉野家・すき家・松屋）、回転寿司（スシロー・くら寿司・はま寿司）、ファミレス（サイゼリヤ・ガスト・ジョイフル）、うどん/そば（丸亀製麺・はなまる）、カフェ（スターバックス・コメダ珈琲・ドトール）、中華（餃子の王将・日高屋・バーミヤン）、ラーメン（一蘭・天下一品）、定食/その他（大戸屋・やよい軒・いきなりステーキ・鳥貴族・サイゼ・幸楽苑）など。国内中心、海外発でも日本で身近なものは可。
- 食べる側の興味（名物メニュー・なぜ安い/速い/うまい）と、**ビジネスの仕組み**（出店戦略・原価・オペレーション・セントラルキッチン・FC か直営か）の両面を扱う。
- ゴールは「kota が次にその店に入ったとき、**裏側の仕組みを思い浮かべながら食べられる**」こと。単なるメニュー紹介や食レポでは終わらせない。

## STEP B. テーマ選定（毎日1チェーン）

- 実在の飲食チェーンを1つ選ぶ。**知名度が高く、深掘りすると意外性のあるもの**を優先。
- **巡回の目安**（ジャンルが偏らないように）：ファストフード → 牛丼 → 回転寿司 → ファミレス → カフェ → 麺類（うどん/そば/ラーメン）→ 中華 → 定食/居酒屋系 → その他（ステーキ・カレー・ピザ・ドーナツ等）、を緩く回す。
- **優先順位**：①誰もが知っていて「へぇ」がある（行ったことある身近さ＋意外な裏側）②なぜ成長したか／なぜ他は真似できないかが語れる ③旬（新業態・話題・再上場・V字回復・不振などニュース性があれば加点）。
- ⚠️ kota が「その店のことは知っていても、**裏の仕組みは知らない**」ものを狙う。「サイゼが安い」だけでは退屈。「なぜサイゼだけがあの価格で利益を出せるのか（自社工場・物流・メニュー設計・イタリア直営農場…）」まで連れて行く。
- `data/gaishoku_history.json` の既出チェーンと被らないこと。

## STEP D. リサーチの観点（RADIO_BASE のリサーチに加えて）

1. **正体と現在地**：創業（いつ・誰が・どこで）、店舗数、売上規模、運営会社・親会社（裏取れる数字だけ）。
2. **成長の仕組み（深掘りの核）**：なぜ伸びたか。出店戦略・立地・FC か直営か・セントラルキッチン・原価率・回転率・人時生産性など、**なぜ可能か／なぜ他社は真似できないか**まで。
3. **名物・看板メニューと、その裏の工夫**：定番商品が生まれた経緯、価格の理由、オペレーション上の妙。
4. **転機・危機**：不振・事件・業態転換・V字回復などの物語。
5. **面白い小ネタ**：常連でも知らない雑学（メニューの裏設定・店舗デザイン・社内文化など）。

## STEP F. 台本の作り（RADIO_BASE の構成に沿って）

- 導入：誰もが知る入り口（「あの黄色いМ」「100円の寿司」など）か、最新の話題（事実ベース）。
- まず「今日のチェーン」を一言で（正体・現在地の数字を最小限）。
- トピック3つは「①どう生まれ、どう広がったか ②なぜ強いのか＝仕組みの核（なぜ他は真似できないか）③名物・転機・小ネタ」のように、**構造が持ち帰れる**粒度で。食レポで終わらせない。
- おわりに：**「次にその店に入ったら、ここを見てみて」**という視点を1つ。
- 数字（店舗数・創業年・価格）は具体的に、裏が取れたものだけ（`_meta` の説明欄に主な出典名）。

### 冒頭・結びの定型
- 冒頭：「こんにちは。外食ラジオ、今日のチェーンは『○○』です。」
- 結び：「以上、今日の外食ラジオでした。次に『○○』に入ったら、ちょっと裏側を、のぞいてみてください。」
